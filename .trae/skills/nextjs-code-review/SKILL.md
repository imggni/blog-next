# Next.js代码审查技能（Trae Skill）

## 审查范围（按优先级）
1. 规范合规：检查是否遵循nextjs-specific规则（TS类型、路由规范、组件命名、样式规范等）
2. 语法错误：检查TypeScript语法错误、类型不匹配、空安全问题、客户端/服务端标识错误
3. 路由问题：检查路由结构是否符合App Router约定、动态路由参数获取是否规范、路由组使用是否正确、page.tsx/layout.tsx配置是否合规
4. 性能问题：检查图片未用next/image优化、未做动态导入、路由懒加载缺失、水合错误隐患、过度重渲染等
5. 安全问题：检查敏感信息硬编码、环境变量使用不当、API路由未做参数校验、服务端组件操作客户端DOM等
6. 最佳实践：检查组件拆分合理性、资源加载规范、依赖使用合规性、静态资源存放路径是否正确

## 输出要求
- 问题清单：行号、问题描述、严重级别（致命/警告/提示）、修复建议、代码示例
- 评分：1-10分，优化建议优先级排序
- 自动修复：可直接应用的修复代码（简单规范问题，如命名错误、缺少类型定义、路由配置错误），复杂问题给出详细步骤
- 总结：明确代码整体合规性，标注需重点优化的模块（如路由结构、性能优化、安全隐患）
