# Git提交规则技能（Trae Skill）

## 指令（Trae执行规则）
1.  接收提交类型、提交描述、修改范围（可选）、详细说明（可选），生成符合Angular规范的Git提交信息
2.  严格校验提交格式，禁止不规范提交（如无类型、描述过短、格式错误），自动修正格式问题
3.  结构固定：类型(范围): 简短描述（≤50字符）→ 空行 → 详细说明（可选，每行≤72字符）→ 空行 → 关联issues（可选）
4.  自动匹配项目技术栈（Next.js/Express），生成贴合开发场景的提交描述，无需手动调整
5.  支持批量校验历史提交信息，给出不规范项及修复建议，支持一键修复

## 提交类型规范（强制）
| 类型       | 说明                                  | 适用场景（贴合Next.js/Express）                  |
|------------|---------------------------------------|-------------------------------------------------|
| feat       | 新增功能                              | 新增Next.js组件、Express接口、路由等            |
| fix        | 修复bug                               | 修复组件渲染异常、接口报错、路由跳转问题等      |
| docs       | 文档修改                              | 更新规则文档、接口文档、注释说明等              |
| style      | 代码样式调整（不影响功能）            | 格式化代码、调整缩进、修改变量命名（非逻辑变更）|
| refactor   | 代码重构（不新增功能、不修复bug）     | 组件拆分、接口逻辑优化、工具函数重构            |
| perf       | 性能优化                              | 图片优化、接口耗时优化、路由懒加载优化         |
| test       | 新增/修改测试用例                     | 编写Jest单元测试、接口测试用例                 |
| chore      | 构建/依赖/配置相关（不影响业务逻辑）  | 升级依赖、修改配置文件、调整构建脚本           |
| revert     | 回滚之前的提交                        | 回滚错误提交、恢复历史版本                     |

## 示例输出模板（基础提交）
```
feat(nextjs): 新增用户卡片客户端组件
```

## 示例输出模板（完整提交）
```
fix(express): 修复登录接口参数校验失败问题

1. 修正zod校验schema，补充userId字段正则校验规则
2. 优化异常提示文案，明确参数错误原因
3. 补充参数校验失败的单元测试用例

Closes #123
```

## 校验与修复规则
1.  格式校验：必须符合「类型(范围): 描述」格式，范围可选（如无明确范围可省略），描述首字母小写、无标点
2.  长度校验：简短描述≤50字符，详细说明每行≤72字符，避免过长提交信息
3.  类型校验：仅允许上述指定类型，禁止自定义类型（如"add"、"update"）
4.  自动修复：对格式错误、类型错误、长度超标问题，自动修正并提示修改点；对描述模糊问题，给出优化建议

## 补充说明
- 范围建议：Next.js相关提交填nextjs，Express相关填express，通用修改填common
- 详细说明：仅在提交内容复杂、需补充说明时添加，简单提交可省略
- 关联issues：使用Closes #xxx（关闭issues）、Related #xxx（关联issues）格式，放在提交信息末尾
